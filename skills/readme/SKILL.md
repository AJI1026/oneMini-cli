# README

## 流程

1. read 项目关键文件：`Cargo.toml` / `package.json` / 入口代码 / 现有 README
2. 识别：项目是什么、谁用、如何安装、如何运行、如何贡献
3. 用**中文简体**撰写（代码与命令保持原样）

## 推荐结构

```markdown
# 项目名

一句话说明。

## 功能

- ...

## 快速开始

### 环境要求
### 安装
### 运行

## 配置（如有）

## 开发

## 许可证
```

## 原则

- 命令可复制粘贴、路径真实存在
- 不编造未实现的特性
- 安装步骤与仓库实际一致（cargo / npm / docker 等）
- 简洁优先，避免 marketing 空话
