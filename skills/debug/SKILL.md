# Debug

## 流程（严格按序）

### 1. 复现

- 获取**精确**错误信息（堆栈、日志、HTTP 状态码）
- 运行**最小复现命令**（单测 / curl / 脚本）
- 确认：稳定复现 vs 偶发

### 2. 定位

- 从堆栈/日志最内层有效帧开始
- 用 grep/read 追踪调用链与数据流
- 形成**可证伪假设**（「若 X 则 Y」）

### 3. 验证假设

- 加日志、断点式 print、或最小实验性改动
- **一次只验证一个假设**

### 4. 最小修复

- 改根因，不做无关重构
- 说明为何此修复正确

### 5. 回归验证

- 重跑复现命令
- 运行相关测试套件
- 报告：根因 / 修复 / 验证结果

## 输出结构

```
## 现象
## 根因
## 修复
## 验证
```

## 禁止

- 未复现就猜测改代码
- 一次改多处 unrelated 文件
- 跳过验证步骤
